#===========================================================================
# 
# Status code -> text data base.
# 
# This file is "compiled" into "stcode.db" using the "mkedb" tool.
# The general structure of this file is a series of:
# 
#     subsys <subsystem number> <subsystem name>
#     ...
#     end
# 
# where the "..." is a series of:
# 
#     module <module number> <module name>
#     ...
#     end
# 
# where the "..." is a series of:
# 
#     code <code number> <code text>
# 
# All numbers are in hex.  Within a given module, all the codes must be
# in ascending order.  (For maintaining ease, the subsystems should be
# kept in order and the modules within a single subsystem should be kept
# in order but "mkedb" does not require this.)
# 
# Blank lines and lines beginning with "#" are ignored.
#
